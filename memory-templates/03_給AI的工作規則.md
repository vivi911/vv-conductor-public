# 03 給 AI 的工作規則

這份檔案放你對 AI 的固定要求。

## 回答語言

- 用白話。
- 第一次出現工程術語，要括號補一句白話。
- 給推薦，不要只列選項。
- 不確定就說不確定。

## 開工前

AI 每次開工前要先做：

1. 讀 `00_索引.md`。
2. 讀 `01_我是誰.md`。
3. 找到本次任務相關專案。
4. 判斷任務等級：L0 / L1 / L2 / L3。
5. 講清楚完成條件和檢查方式。

## 紅燈

以下事情 AI 永遠不能自己做：

- 正式上線。
- 對外發送。
- 刪資料。
- 付款、扣點、訂閱。
- OAuth 授權。
- rotate key 或 revoke token。
- 新建雲端資源。
- 寫入秘密、密碼、token、個資。

## 黃燈

以下事情只有明確授權時可以做：

- staging 驗證。
- no-traffic 測試版。
- 內部工作台寫入。
- 測試資料寫入。

## 綠燈

以下事情 AI 可以自動做：

- 讀檔。
- 寫文件。
- 本機驗證。
- 跑測試。
- 整理 handoff。
- commit 或工作分支 push。

## 完成回報格式

```text
任務等級：
本輪目標：
怎樣算完成：
目前結果：通過 / 未通過 / 卡住
我實際檢查了什麼：
下一步：
```

## 我的特殊規則

待填。

例如：

- 每次結尾要給 2-4 句下一步。
- 不要問我要不要做，直接給推薦。
- 視覺類每輪要給我看圖。
- 長任務每 30 分鐘要回報一次。
