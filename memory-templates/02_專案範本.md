# 02 專案範本

每個專案複製一份這個格式。

## 專案名稱

待填。

## 這個專案在解決什麼問題

待填。

用一句白話說清楚：

```text
這個專案是為了讓誰，在什麼情境下，少掉什麼麻煩，得到什麼結果。
```

## 目前做到哪

待填。

請寫具體狀態，不要只寫「進行中」。

## 目前卡點

待填。

卡點分三種：

- 決策卡住：需要人拍板。
- 資料卡住：缺資料、缺權限、缺來源。
- 施工卡住：技術或執行問題。

## 下一步最推薦的 3 張卡

1. 待填。
2. 待填。
3. 待填。

## 需要我拍板的點

待填。

## 不能碰的範圍

待填。

例如：

- 不碰正式會員。
- 不發外部通知。
- 不切正式流量。
- 不刪資料。

## 真相來源

| 類型 | 路徑或連結 |
|---|---|
| 最新 handoff | 待填 |
| 規則文件 | 待填 |
| 主要資料 | 待填 |
| 交付物 | 待填 |

## 給 AI 的開工提示

```text
請先讀本專案摘要，再讀真相來源。
先回報：目前做到哪、卡點、下一步推薦。
沒有證據不要說完成。
```
